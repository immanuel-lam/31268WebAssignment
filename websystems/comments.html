Design Comments

This page covers the design decisions I made when building this site.

a) Overall Website Structure

The site has four pages:

  • index.html - home page, intro and overview
  • past.html - computing background
  • future.html - career goals
  • comments.html - this page, design reflection

All four pages link to each other through the nav bar at the top. You can get anywhere from anywhere without hitting back.

All styles are in a single external file, websystems.css, linked from every page with:

<link rel="stylesheet" type="text/css" href="websystems.css">

Site map showing how all the pages connect:

b) Internal Page Structure

Every page uses the same structure:

<div id="page-wrap">             -- outer container, 860px, centred
    <header class="site-header"> -- title and subtitle
    <nav class="top-nav">        -- navigation
    <div class="main-content">   -- page content
    <footer class="page-footer"> -- footer
</div>

I used semantic HTML5 elements (header, nav, footer) because they actually describe what the content is. The main content area uses a div since it's a generic container and that's what div is for.

index.html has the intro-block div, a photo, and two lists. past.html has a timeline table and two lists. future.html has a certifications table and skill lists. comments.html (this page) is the longest, mostly the big selector table and the design sections.

d) Graphic Design

Colours

I went with dark teal as the main colour. Blue is really common for this kind of site and I wanted something a bit different. The colours used:

  • #1f5252 - dark teal, header background, h2 headings, table headers, link colour
  • #163d3d - darker teal, nav bar and footer background
  • #2e7a7a - mid teal, nav hover and current page highlight
  • #99cccc - light teal, header subtitle text
  • #ebebeb - light grey, body background
  • #ffffff - white, page content background
  • #1a1a1a - near-black, body text

The same dark teal is used across the header, h2 headings, table headers and links so everything looks like it belongs together. The nav bar and footer use a slightly darker version of it. I checked the colour combinations against the bad pairings list in the spec - none of those are used.

Fonts

Arial throughout. No external fonts needed and it works on every system. Size hierarchy:

  • h1 in header: 22px
  • nav links: 14px
  • h2 section headings: 1.2em
  • h3 sub-headings: 1em
  • body text: inherited (browser default 16px)
  • footer and tables: 13px

Layout

860px fixed width, centred on the page. White content box on a light grey body so the page has some depth. The header and the whole page are left-aligned so the h1 sits to the left, not centred - I think it looks a bit less generic that way.

Structure is the same on every page: header, nav, content, footer. Paragraph text is all left-aligned.

What worked

Using one colour consistently across headers, headings, links and table headers makes the page look consistent without having to do anything complicated. The teal feels a bit more personal than just using blue.

What could be better

The site is simple. The content pages are mostly just text and lists. I prioritised getting the structure right over making it visually interesting, which I think was the right call for this assignment.

e) Accessibility

Alt Text

The image on index.html:

<img src="images/me.png" alt="A photo of me and friends playing pool">

Photo is mine so no credit needed. The alt text describes what the image shows. Screen readers read this aloud and it also shows if the image fails to load.

The site map image on this page also has alt text describing what it shows.

CSS Off Test

Tested in Firefox with CSS disabled (View, Page Style, No Style). With CSS off:

  • All text content is still readable
  • Nav links at the top still work so you can still move between pages
  • Tables still display as normal HTML tables
  • Lists and headings keep their structure
  • Reading order (header, nav, content, footer) still makes sense

Using HTML5 semantic elements (header, nav, footer) means the structure is meaningful without CSS, not just visual.

Other

  • Links have descriptive text, not just "click here"
  • No JavaScript - works with JS disabled
  • Good contrast throughout: dark text on white, light text on dark backgrounds
  • No horizontal scrolling at normal viewport sizes
  • No animations

Design Comments

This page goes through the design decisions I made when building this site.

a) Overall Website Structure

The site has four pages:

  • index.html: home page, introduces me and the site
  • past.html: my computing history and background
  • future.html: career goals and future plans
  • comments.html: this page, design reflection

All four pages are linked together through the navigation bar at the top of every page. You can get to any page from any other page without needing the back button.

The CSS is in a single external stylesheet called websystems.css, linked from every page using:

<link rel="stylesheet" type="text/css" href="websystems.css">

This means all four pages share the same styles. Any style change only needs to be made in one place.

b) Internal Structure of Each Page

Every page follows the same basic structure:

<div id="page-wrap">            -- outer container, sets the width to 900px
    <header class="site-header"> -- site title and subtitle
    <nav class="top-nav">        -- navigation links
    <div class="main-content">   -- main page content
    <footer class="page-footer"> -- footer line
</div>

I used semantic HTML5 elements (header, nav, footer) because they describe what the content actually is. For the main content area I used a div with a class since the rubric required at least one div. It also helps screen readers and keeps the HTML easier to read.

The index.html page has a heading, an intro-block div, an image, and two unordered lists plus a short paragraph with links to the other pages. past.html has several sections with paragraphs, a timeline table, and two lists. future.html is similar to past.html with a table for certifications and lists for goal areas. comments.html (this page) is the longest, mostly prose sections and one big CSS table.

d) Graphic Design Discussion

Colour Choices

I went with dark navy because it felt right for a cybersecurity site. The colours I used:

  • #1b3a5c: dark navy, used for the header and h2 headings
  • #12293f: darker navy, used for the nav bar and footer
  • #1e6fa0: medium blue, accent colour for links and the intro block border
  • #dddddd: light grey, body background behind the white content area
  • #ffffff: white, main content background
  • #111111: near-black, body text

The light grey body background with a white content area in the middle makes the structure visible without being distracting. The blue accent is used consistently across the nav hover, the intro block border, links and h2 borders so the page feels like it belongs together without being over-designed.

I also made sure none of the colour combinations are in the bad pairings list from the brief (red on black, purple on black etc.).

Font Choices

I used Arial throughout (no fallback needed on most systems). Sizes:

  • h1 in header: 26px
  • Nav links: 14px
  • Section headings (h2): 1.3em
  • Body text: inherited (16px browser default)
  • Footer and table text: 13px

I used a pixel size for the header h1 since it's a fixed element that shouldn't scale with the surrounding text. The h2 headings use em so they stay proportional to the body text size.

Layout

I went with a single column layout set to 900px wide and centred on the page. I used a fixed width rather than max-width so the layout doesn't change depending on the screen width - it's simpler to predict how it looks. The structure is always header, nav, content, footer in that order on every page.

Paragraph text is left-aligned. Headings in the header are centred but everything in the content area is left-aligned, which is easier to read for longer text.

What I think worked

Using the same dark navy for the header and the section heading colour makes the page feel consistent without needing to be clever about it. The intro block on the home page with the blue left border is a small touch that I think makes that section stand out without looking out of place.

What could be improved

The layout is pretty basic. I kept it simple on purpose because I wanted to get the structure right first, but I could have done more with spacing and maybe added a bit more visual interest to the content pages.

e) Accessibility

Alt Attributes

The image on index.html has an alt attribute:

<img src="images/me.png" alt="A photo of me and friends playing pool">

The photo was taken by me so no external credit is needed. The alt text describes what the image actually shows. This matters because screen readers read it aloud for visually impaired users and it also shows up if the image fails to load.

The site map image on this page also has an alt attribute describing what it shows.

CSS Disabled Test

I tested the site in Firefox with CSS turned off (View, Page Style, No Style). With CSS off:

  • All content is still readable, it just loses its colours and formatting
  • The navigation links at the top of each page are still plain text links you can click, so you can still move between all four pages
  • The tables on the past and future pages still render as normal HTML tables
  • Lists, headings and paragraphs still have the right logical structure
  • The page still makes sense as plain HTML. The order of elements (header, nav, content, footer) still reflects the reading order

Using semantic HTML5 elements (header, nav, footer) means the structure is meaningful without CSS, not just visual grouping.

Other Accessibility Considerations

  • Links use descriptive text rather than just "click here"
  • No JavaScript on the site so it works with JS disabled
  • Contrast is high throughout (dark text on white, white text on dark backgrounds)
  • No horizontal scrolling at normal window sizes
  • No flashing animations or effects

Design & Structure Comments

This page reflects on the design decisions, HTML structure, and CSS used throughout the site. It is written as part of the UTS Web Systems assignment requirements.

Overall Site Structure

The site consists of four HTML pages, all sharing a single stylesheet (websystems.css) and a consistent navigation bar. The four pages are:

  • index.html — Home/intro page with a welcome message and site overview.
  • past.html — Background page covering homelabbing, networking, self-hosted services, and web development.
  • future.html — Goals page focused on red team cybersecurity ambitions.
  • comments.html — This page, documenting the site's structure, styles, and design rationale.

Navigation is provided on every page via a <nav> element containing an unordered list of links. No Back button is ever needed. All internal links use relative paths. There are no frames, iframes, or external frameworks.

Per-Page HTML Structure

index.html
Contains a <nav class="nav-bar"> with a <ul> of four links, a <div id="main-content"> wrapping an <h1> with an inline <span class="highlight"> for the name accent, three <section class="section-block"> elements, and a <footer>.

past.html
Shares the same nav and footer pattern. Contains four <section class="section-block"> elements covering homelabbing, networking, self-hosted services, and web development. The self-hosted services section includes a <table class="service-table"> with a <caption>, <thead>, and <tbody> using scope attributes on header cells. Also contains an ordered list of web development skills and a <figure> with an <img> and <figcaption>.

future.html
Contains an intro paragraph and three <section class="section-block"> elements. The "Areas of Interest" section uses a <dl> definition list with six <dt>/<dd> pairs. An unordered list covers current steps being taken.

comments.html
This page. Contains five <section class="section-block"> elements with descriptive text, two HTML tables (styles table and structure notes), and an unordered list. All content is authored in semantic HTML with no inline styles.

Graphic Design Rationale

Colour palette: The background is off-white (#f9f9f9) rather than pure white to reduce eye strain on bright screens. Body text uses near-black (#1a1a1a) for high contrast without the harshness of 100% black. The accent colour is deep teal (#2a7a7a), which provides strong contrast against both the light background and the dark navigation bar while feeling distinct from the generic blues most sites use. Teal was chosen specifically because it reads as technical and deliberate — appropriate for a site about IT and cybersecurity.

Typography: The site uses a system font stack (-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica, Arial, sans-serif). This means no external font requests, faster load times, no dependency on third-party CDNs, and a familiar, legible reading experience on every platform. Body text is set at 1rem (browser default, typically 16px) with a line-height of 1.7 to give text room to breathe.

Layout and whitespace: Content is constrained to a 900px maximum width and centred with auto margins. This prevents uncomfortably long line lengths on wide monitors. Paragraphs are further limited to 72 characters via max-width: 72ch, which is within the typographic sweet spot of 60–75 characters per line. Generous section spacing (margin-top: 2.5rem) separates distinct content areas without needing dividers or decorative elements.

Navigation: The nav bar uses a dark background (#1a1a1a) that contrasts strongly with the page. The active page link is visually differentiated using the teal accent colour and a bottom border, giving users a clear sense of where they are in the site at all times.

Accessibility

Alt text: Every image on the site has a descriptive alt attribute. The homelab diagram on past.html has alt text that describes the content of the image so that users who cannot see the image — including screen reader users and those browsing with images disabled — still understand what is being represented.

Semantic structure: Pages use semantic HTML5 elements: <nav>, <main>, <section>, <figure>, <figcaption>, <footer>, and <dl>/<dt>/<dd>. Heading levels are used in strict hierarchical order (h1 → h2). Tables use <thead>, <tbody>, <caption>, and scope attributes on <th> cells so that screen readers can announce column context correctly.

CSS disabled: With CSS turned off, the site degrades gracefully. All content remains fully readable and navigable because it is structured in semantic HTML. The navigation becomes a plain vertical list of links, headings are rendered in the browser's default heading styles, and tables remain tabular. No information is conveyed purely through visual styling — the structure itself carries the meaning.

Keyboard navigation: All interactive elements (links) are reachable via Tab key. Focus states are styled with the same teal accent as hover states so that keyboard users receive the same visual feedback as mouse users.

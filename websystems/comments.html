Design Comments

This page goes through the design decisions I made when building this site.

a) Overall Website Structure

The site has four pages:

  • index.html: home page, introduces me and the site
  • past.html: my computing history and background
  • future.html: career goals and future plans
  • comments.html: this page, design reflection

All four pages are linked together through the navigation bar at the top of every page. You can get to any page from any other page without needing the back button.

The CSS is in a single external stylesheet called websystems.css, linked from every page using:

<link rel="stylesheet" type="text/css" href="websystems.css">

This means all four pages share the same styles. Any style change only needs to be made in one place.

b) Internal Structure of Each Page

Every page follows the same basic structure:

<div id="main-wrapper">           -- outer container, limits width to 960px
    <header class="page-header">  -- site title and subtitle
    <nav class="nav-bar">         -- navigation links
    <main class="content-area">   -- main page content
    <footer class="footer-text">  -- footer line
</div>

I used semantic HTML5 elements (header, nav, main, footer) because they describe what the content actually is rather than just grouping things visually. It also helps screen readers and keeps the HTML easier to read.

The index.html page has a heading, an intro-box div, an image, and two unordered lists plus a short paragraph with links to the other pages. past.html has several sections with paragraphs, a timeline table, and two lists. future.html is similar to past.html with a table for certifications and lists for goal areas. comments.html (this page) is the longest, mostly prose sections and one big CSS table.

d) Graphic Design Discussion

Colour Choices

I went with dark navy and blue because it felt right for a cybersecurity site without going full hacker aesthetic (no green on black). The main colours are:

  • #2c3e50: dark slate blue, used for the header and table headers
  • #1a252f: darker navy, used for the nav bar and footer
  • #2980b9: medium blue, accent colour for links, hovers and borders
  • #f0f0f0: light grey, page background behind the content
  • #ffffff: white, main content background
  • #333333: near-black, body text

The dark header with a white content area makes the structure obvious straight away. The blue accent colour ties the nav hover effects, heading underlines, and link colours together. I also made sure none of my colour combinations were in the bad pairings list from the brief (things like red on black or purple on black).

The light blue background on the intro box (#eaf3fb) was a small touch but I think it works well without being too loud.

Font Choices

I used Arial throughout (Helvetica as a fallback). It's clean, works at every size, and doesn't need any external fonts loading. The size hierarchy is:

  • h1 in header: 2.2em
  • Nav links: 1.05em
  • Section headings (h2): 1.5em
  • Body text (p): 1em (inherited)
  • Footer: 0.82em

That follows the hierarchy from the brief. The h2 headings inside the content are 1.5em which is bigger than the nav links at 1.05em, but since the main h1 at 2.2em is clearly the largest thing on the page it still feels right.

Layout

I went with a single column layout centred on the page with a max-width of 960px. On wide screens the content doesn't stretch out to the full viewport which would make the lines too long to read. The structure is always header, nav, content, footer in that order on every page.

I didn't centre the body text. Headings are centred in the header but all paragraph text is left-aligned which is easier to read for longer sections.

What I think worked

Making the nav slightly darker than the header felt cleaner than having them match. The blue bottom border on section headings is simple but adds some structure.

What could be improved

The site is pretty minimal. I could have made it more interesting with more time. The photo on the home page is also just a placeholder, I'd swap it out for a real one before submitting.

e) Accessibility

Alt Attributes

The image on index.html has an alt attribute:

<img src="images/me.png" alt="A placeholder image">

For the actual submission this would describe the real photo, something like "A photo of me at the UTS campus". The alt text matters because screen readers read it aloud for visually impaired users and it also shows up if the image can't load.

CSS Disabled Test

I tested the site in Firefox with CSS turned off (View, Page Style, No Style). With CSS off:

  • All content is still readable, it just loses its colours and formatting
  • The navigation links at the top of each page are still plain text links you can click, so you can still move between all four pages
  • The tables on the past and future pages still render as normal HTML tables
  • Lists, headings and paragraphs still have the right logical structure
  • The page still makes sense as plain HTML. The order of elements (header, nav, content, footer) still reflects the reading order

Using semantic HTML elements (header, nav, main, footer) means the structure is meaningful without CSS, not just visual grouping.

Other Accessibility Considerations

  • Links use descriptive text rather than just "click here"
  • No JavaScript on the site so it works with JS disabled
  • Contrast is high throughout (dark text on white, white text on dark backgrounds)
  • No horizontal scrolling at normal window sizes
  • No flashing animations or effects

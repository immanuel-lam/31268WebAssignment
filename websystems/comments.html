Design Comments

This page goes through the design decisions I made when building this site.

a) Overall Website Structure

The site has four pages:

  • index.html: home page, introduces me and the site
  • past.html: my computing history and background
  • future.html: career goals and future plans
  • comments.html: this page, design reflection

All four pages are linked through the navigation bar at the top of every page. You can get to any page from any other page without using the back button.

The CSS is in one external file called websystems.css, linked from every page with:

<link rel="stylesheet" type="text/css" href="websystems.css">

This keeps all styles in one place so changing a style automatically affects all pages.

b) Internal Structure of Each Page

Every page uses the same structure:

<div id="page-wrap">             -- outer container, 880px wide, centred
    <header class="site-header"> -- site title and subtitle
    <nav class="top-nav">        -- navigation links
    <div class="main-content">   -- main page content
    <footer class="page-footer"> -- footer
</div>

I used HTML5 semantic elements (header, nav, footer) where the element name actually describes what the content is. The main content area uses a div with a class because the rubric required at least one div, and div is the right element for a generic content container.

index.html has a heading, an intro-block div, a photo, and two unordered lists plus a paragraph linking to the other pages. past.html has several sections with a timeline table and two lists. future.html is similar with a certifications table and lists. comments.html (this page) is the longest page, with a big selector table and several sections.

d) Graphic Design Discussion

Colour Choices

I used a royal blue (#2a5298) as the main colour throughout — for the header, footer, nav links, h2 headings and table headers. I picked it because I wanted a blue that felt a bit more personal than the generic dark navy you see on a lot of websites. All the blue elements use the same colour so the page feels consistent.

The specific colours used are:

  • #2a5298: royal blue, main accent, header, footer, nav links, headings, table headers
  • #adc6e8: light blue, used for the subtitle in the header and footer text
  • #e8e8e8: light grey, body background — sits behind the white content box
  • #ffffff: white, main content background
  • #222222: near-black, body text
  • #f4f4f4: very light grey, nav bar background

The nav bar uses a light grey background with blue text instead of a dark background like the header. This means the page doesn't just have stacked dark bands at the top — there's a visible difference between the header and the nav, which I think looks cleaner.

I checked all my colour combinations against the bad pairings listed in the spec (red on black, purple on black, etc.) — none of those combinations are used.

Font Choices

Arial throughout. I didn't want to pull in any external fonts since the spec says not to rely on third party resources. The size hierarchy is:

  • h1 in header: 24px
  • Nav links: 14px, bold
  • Section headings (h2): 1.25em
  • Sub-headings (h3): 1em
  • Body text (p): inherited (16px browser default)
  • Footer and table text: 13px

Layout

Single column layout, 880px wide, centred on the page. The white content box sits on the light grey body background so it stands out as its own box. I added 15px of padding to the body top and bottom so there's a small gap above and below the white box instead of it just running into the edge of the viewport.

The structure on every page is: header, nav bar, content, footer in that order. Text in the content area is left-aligned. The header is centred because it's a title block but for normal reading the left-aligned text is easier.

What I think worked

Having the nav bar light instead of dark makes a real visual difference — it doesn't look like a stacked dark header. The header and footer matching in blue creates a frame around the content which I think looks intentional. Keeping one accent colour throughout makes the site feel consistent without needing to think too hard about colour combinations.

What could be improved

The layout is very simple. I focused on getting the structure right and the styles consistent. With more time I would have spent more effort on the spacing and maybe added something to make the content pages less plain.

e) Accessibility

Alt Attributes

The image on index.html has an alt attribute:

<img src="images/me.png" alt="A photo of me and friends playing pool">

The photo was taken by me so no external credit is needed. The alt text describes what the image actually shows. Screen readers read this aloud for visually impaired users and it also shows up if the image fails to load.

The site map image on this page also has descriptive alt text.

CSS Disabled Test

I tested in Firefox with CSS off (View, Page Style, No Style). With CSS disabled:

  • All content is still readable — it just loses colours and layout
  • The nav links at the top of each page still work as plain text links, so you can still navigate between all four pages
  • Tables on past.html and future.html still render as normal HTML tables
  • Lists, headings and paragraphs keep their logical structure
  • The reading order (header, nav, content, footer) still makes sense as plain HTML

Using semantic HTML5 elements (header, nav, footer) means the page has meaningful structure even without CSS.

Other Accessibility Considerations

  • Links use descriptive text rather than "click here"
  • No JavaScript — site works with JS disabled
  • High contrast throughout: dark text on white, white/light text on blue backgrounds
  • No horizontal scrolling at normal sizes
  • No animations or flashing
